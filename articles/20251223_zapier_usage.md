## はじめに

私が所属している新規事業部アプリチームでは、日々の定型業務を効率化するために Zapier を活用しています。
本記事では、実際に運用している 4 つの自動化事例を紹介します。

## 1. 夕会ファシリテータの自動アサイン

### 課題

毎日の夕会でファシリテータを決める際、「今日は誰がやる？」、「今日のファシリ誰でしたっけ？」という確認作業が発生していました。当番制にしても、手動でのローテーション管理は忘れがちです。

### 解決策

Google Calendar の夕会イベントをトリガーに、開始 10 分前に Slack へファシリテータを通知する仕組みを構築しました。

**Zap の構成：**

1. **Google Calendar** - 夕会イベントの開始 10 分前にトリガー
2. **Formatter by Zapier** - Pick from list を使って参加者のメンバー ID からランダムで 1 つ抽出
3. **Slack** - チャンネルに担当者をメンション付きで通知

![](/images/20251223/facilitator.png)
**Zap の構成（再抽選）：**

アサインされた人が休みなどで参加できない場合に備え、再抽選の仕組みも用意しています。

1. **Slack** - 通知メッセージに特定のリアクション（例：🔄）が付いたらトリガー
2. **Formatter by Zapier** - ランダムに再抽選
3. **Slack** - 同じスレッドに新しい担当者を通知

![](/images/20251223/facilitator_reroll.png)

### 効果

- ファシリテータ決めの時間が 0 になった
- **不正のない筋肉ルーレット**の導入により公平なアサインになった
- 3日連続で当たる人や今日のファシリを予言する人が現れたりしてわいわいとした雰囲気にもなった

## 2. アプリストア公開速報の Slack 通知

### 課題

App Store や Google Play へのアプリ審査提出後、公開されたかどうかを手動で確認する必要がありました。アプリのプッシュ通知やストアからのメールを受け取ることもできますが気づくのに遅れてしまうことがありました。

### 解決策

App Store や Google Play からアプリ公開時に届くメールをトリガーに、Slack へ通知する Zap を作成しました。

**Zap の構成（公開通知）：**

1. **Gmail** - 各ストアからの公開通知メールを受信したらトリガー
2. **Slack** - リリースチャンネルに速報を投稿

![](/images/20251223/apple_notify.png)

**Zap の構成（審査リジェクト通知）：**

1. **Gmail** - 審査リジェクトのメールを受信したらトリガー
2. **ChatGPT** - メール本文を日本語に翻訳・要約
3. **Slack** - リジェクト理由をわかりやすく投稿

![](/images/20251223/android_notify.png)

### 効果

- ストアからメールを受信していないメンバーも結果をすぐに確認できるようになった
- 公開と同時にチーム全体へ共有できるようになった
- リリース作業の完了をリアルタイムで把握できるようになった

## 3. Confluence の議事録自動作成

### 課題

定例ミーティングの議事録作成は地味に手間がかかります。Confluence で新しいページを作成し、テンプレートを適用し、日付やメンバーを入力する作業を毎回行っていました。

### 解決策

Google Calendar の会議イベントをトリガーに、Confluence API で前回の議事録を複製し、作成されたページの URL を Slack に通知する Zap を構築しました。

**Zap の構成：**

1. **Google Calendar** - 特定のカレンダーで会議が開始する前日にトリガー
2. **Webhooks by Zapier** - Confluence API で前回の議事録ページを複製
3. **Slack** - 作成された議事録ページの URL をチャンネルに通知

![](/images/20251223/gijiroku_max.png)

### 効果

- 議事録作成の初期作業が 0 になった
- テンプレートが統一され、議事録の品質が安定した
- 会議前に議事録ページが準備されているため、すぐに記録を開始できる

## 4. 次バージョンのチケット進捗報告 Bot

### 課題

リリースごとにチーム内でリリース担当決めて、コードフリーズまでに全てマージできるか担当者が確認をする運用をしています。
そのために都度Jiraのバージョンを開いて確認をする手間がありました。

### 解決策

Jira の API を使用して次バージョンのチケット状況を集計し、毎日決まった時刻に Slack へレポートを投稿する Zap を作成しました。

**Zap の構成：**

1. **Schedule by Zapier** - 毎日朝 11 時にトリガー
2. **Webhooks by Zapier** - Jira API で次バージョンのチケットを取得、データを整形
4. **Slack** - 進捗レポートを投稿

**投稿されるレポート例：**

![](/images/20251223/jira_progress_bot.png)

### 効果

- 毎朝自動で進捗が可視化されるようになった
- 未アサインのチケット検知ができるようになった
- リリース担当者以外のメンバーも状況把握しやすくなった
- 安心できる

## まとめ

Zapier を活用することで、以下のような定型業務を自動化できました。

| 自動化項目             | 削減できた作業時間（目安） |
| ---------------------- | -------------------------- |
| 夕会ファシリテータ決め | 3 分/日                    |
| ストア公開確認         | 15 分/リリース             |
| 議事録作成準備         | 5 分/会議                  |
| 進捗レポート作成       | 15 分/日                   |

小さな自動化の積み重ねが、チーム全体の生産性向上につながります。「これ自動化できないかな？」と思ったら、まずは Zapier で試してみることをおすすめします。
